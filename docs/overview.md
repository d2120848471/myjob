# 项目概览

## 背景

当前仓库已经从“手写 `net/http + sqlx + redis` 的普通后端”切换为
“仓库根即主应用的 GoFrame 单体后台项目”。
这次重构的核心不是只换一个框架名字，而是把原来堆在 `internal/app` 的启动、
协议、权限、业务、数据和基础能力拆开，让多人协作时更容易定位和演进。

## 当前目标

- 保持后台 HTTP 路由和响应结构尽量兼容旧实现
- 在仓库根形成统一的主应用入口与工程目录
- 把登录鉴权、RBAC、短信、审计、IP 归属地等基础能力沉到专门位置
- 把测试、文档、脚本、SQL、配置文件放到稳定目录，便于团队协作

## 适用范围

本项目当前定位是“一个规范的大型单体 GoFrame 后台”，不做微服务拆分，
也不预埋不必要的平台化基础设施。

## 兼容约束

- 保留 `/api/admin/*` 既有接口路径和方法
- 保留 `code / msg / data` 响应包裹结构
- 继续支持 `ADMIN_CONFIG`、`SUPER_ADMIN_PHONE`、`SUPER_ADMIN_PASSWORD`
  这类运行时入口
- 继续复用 MySQL、Redis、短信配置、菜单权限、日志表等现有业务语义
